Pipeline

From webcam frame to lit LED, six steps

Every loop of the program runs the same short chain — nothing is stateful beyond a small smoothing buffer, so it recovers instantly if a hand leaves the frame.

01

Capture

OpenCV grabs a frame from the webcam via VideoCapture, mirrored for a natural view.

02

Detect

MediaPipe Hands locates the hand and returns 21 landmark points, up to 60 times a second.

03

Count

Each fingertip's position is compared to its lower joint to decide if that finger is raised.

04

Filter

A short rolling buffer of recent counts smooths out single-frame flicker before it reaches the LEDs.

05

Transmit

The stable count is written as one character over serial with PySerial, at a fixed baud rate.

06

Actuate

The Arduino sketch reads that byte and drives digitalWrite on the matching LED pins.

The two halves of the system

A short Python loop does the seeing and counting; a short Arduino sketch does the lighting.

finger_count.pyPython
def count_fingers(landmarks):
    tip_ids = [4, 8, 12, 16, 20]
    fingers = []

    # thumb: compare x, it folds sideways
    if landmarks[4].x < landmarks[3].x:
        fingers.append(1)
    else:
        fingers.append(0)

    # other four: tip above its pip joint = up
    for tip in tip_ids[1:]:
        if landmarks[tip].y < landmarks[tip - 2].y:
            fingers.append(1)
        else:
            fingers.append(0)

    return sum(fingers)
led_driver.inoArduino
const int ledPins[5] = {2, 3, 4, 5, 6};

void setup() {
  Serial.begin(9600);
  for (int i = 0; i < 5; i++) {
    pinMode(ledPins[i], OUTPUT);
  }
}

void loop() {
  if (Serial.available()) {
    int count = Serial.parseInt();
    for (int i = 0; i < 5; i++) {
      digitalWrite(ledPins[i], i < count ? HIGH : LOW);
    }
  }
}
